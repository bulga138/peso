# PESO Benchmarking

## Goal

Measure whether PESO's enhancements improve model output quality while reducing token costs.

## Metrics

| Metric             | What it measures                                         | How                                           |
| ------------------ | -------------------------------------------------------- | --------------------------------------------- |
| **Token delta**    | Input tokens: enhanced vs original                       | Count tokens before/after pipeline            |
| **Score delta**    | PESO score improvement                                   | `peso-score` before vs after                  |
| **Output quality** | Does the model produce better code?                      | Manual review + test pass rate                |
| **Cost ratio**     | Enhancement cost vs savings from better first-try output | (small model tokens) / (avoided retry tokens) |

## Test Corpus

Create test prompts in `benchmarks/prompts/`:

```
benchmarks/
  prompts/
    code-simple.txt        # "fix the login bug"
    code-complex.txt       # multi-file refactor description
    research-stale.txt     # "what's the latest React version"
    creative-short.txt     # "write a blog post about X"
    ambiguous.txt          # "make it better"
    already-good.txt       # well-structured prompt (should passthrough)
  results/
    <auto-generated>
```

## Running Benchmarks

### Manual

```bash
# Score a single prompt
echo "fix the login bug" | bun run src/cli.ts score

# Enhance and compare
echo "fix the login bug" | bun run src/cli.ts enhance --mode code

# Full trace
echo "fix the login bug" | bun run src/cli.ts debug
```

### With taco (@bulga138/taco)

Integration with [taco](https://github.com/bulga138/taco) for automated benchmark runs:

```bash
# Install taco plugin
# (add to opencode.json plugins or install globally)

# Run benchmark suite
taco run benchmarks/prompts/ --evaluator peso-score --output benchmarks/results/

# Compare runs
taco diff benchmarks/results/run-001.json benchmarks/results/run-002.json
```

#### taco Evaluation Hook

PESO exposes a scoring function that taco can call:

```typescript
import { scorePrompt } from 'peso/scorer';
import { classify } from 'peso/classifier';
import { runPipeline } from 'peso/enhancer';

// Score original
const before = scorePrompt(prompt);

// Enhance
const classification = classify(prompt, 'auto');
const result = runPipeline(prompt, classification, {
  domain: classification.domain,
  injectContext: false,
});

// Score enhanced
const after = scorePrompt(result.enhanced);

// Report
console.log(`${before.total} → ${after.total} (Δ${(after.total - before.total).toFixed(1)})`);
```

## Key Questions

1. **Does PESO help simple prompts?** — Should passthrough kick in more aggressively?
2. **Does LLM rewrite justify its cost?** — Compare rule-based-only vs rule+LLM
3. **Does context injection help or bloat?** — Test with/without `<peso:context>`
4. **Do tool priorities reduce unnecessary tool calls?** — Count tool invocations with/without priorities

## Baseline Protocol

1. Run 20 prompts through main model **without** PESO
2. Run same 20 prompts **with** PESO (rule-based only, `use_llm: false`)
3. Run same 20 prompts **with** PESO (full pipeline, `use_llm: true`)
4. Compare: output quality, token counts, time-to-correct-answer
